Application Initialization Module

Overview

This module sets up the FastAPI application with comprehensive configuration, middleware, exception handling, and core routes.

Key Components

Application Lifespan Management

lifespan(app: FastAPI)

Purpose: Manage application startup and shutdown lifecycle

Key Operations: - Establish database connection using AsyncIOMotorClient - Log application start and database connection status - Gracefully close database connection on app exit - Handle and log any connection errors

Exception Handlers

http_exception_handler(request: Request, exc: HTTPException)

Purpose: Handle HTTP-specific exceptions - Manages CORS headers - Logs exception details - Returns JSON response with error information

client_server_exception_handler(request: Request, exc: Exception)

Purpose: Handle various types of exceptions

Supported Exception Types: - ClientException: Custom client-side errors - ValidationError: Pydantic validation errors - ExpiredSignatureError: JWT token expiration - TokenException: Authentication token issues - Generic server errors

Routes

root()

  • Endpoint: /
  • Returns simple health check message

version()

  • Endpoint: /version
  • Returns current application version from pyproject.toml

changelog(request: Request, user: TokenDecrypted)

  • Endpoint: /changelog
  • Requires user authentication
  • Renders CHANGELOG.md as HTML

Configuration Features

  • CORS middleware configuration
  • Dynamic origin handling
  • Centralized logging
  • Version management from TOML configuration

Dependencies

  • FastAPI
  • Motor (Async MongoDB)
  • JWT
  • Markdown
  • TOML
  • Starlette

Environment Setup

  • Reads configuration from pyproject.toml
  • Connects to MongoDB using URI from settings
  • Supports dynamic CORS origin configuration

Logging

  • Uses custom logging mechanism
  • Tracks trace IDs for error tracking
  • Logs application events and errors

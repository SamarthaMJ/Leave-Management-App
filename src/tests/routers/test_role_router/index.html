Role Router Tests

Overview

This module contains unit tests for the role router, covering CRUD (Create, Read, Update, Delete) operations for roles.

Helper Functions

get_role()

Creates a sample role with: - Randomly generated ObjectId - Name: "Level 1" - Permissions: read_only

Test Cases

test_role_get_all

Purpose: Verify retrieval of all roles

Steps: - Send GET request to "/roles" endpoint - Verify: - Response status code is 200 (OK) - Response is a list

Parameters: - client: AsyncClient for making HTTP requests - db: Async MongoDB database connection

test_role_post

Purpose: Validate role creation

Steps: - Create a new role using get_role() - Send POST request to "/roles" endpoint - Verify: - Response status code is 201 (Created) - Returned role matches the created role

Parameters: - client: AsyncClient for making HTTP requests

test_role_update

Purpose: Test updating an existing role

Steps: - Create a new role - Update role name to "Level 2" - Send PATCH request to role update endpoint - Verify: - Response status code is 200 (OK) - Role is updated correctly

Parameters: - client: AsyncClient for making HTTP requests

test_role_delete

Purpose: Validate role deletion

Steps: - Create a new role - Send DELETE request to role deletion endpoint - Verify: - Response status code is 200 (OK) - Deletion status is True

Parameters: - client: AsyncClient for making HTTP requests

Dependencies

  • pytest
  • bson
  • httpx

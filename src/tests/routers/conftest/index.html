Test Suite Documentation

Configuration (conftest.py)

Purpose

Provides pytest fixtures and configuration for testing the application with a test database.

Key Features

  • Sets up test environment variables
  • Configures MongoDB test database
  • Creates test fixtures for:
  • Database connection
  • Test role
  • Test user
  • AsyncClient for API testing

Test Environment Setup

  • Uses a separate test database
  • Drops and recreates collections for each test session
  • Overrides authentication for testing

Authentication Router Tests (test_auth_router.py)

Test Scenarios

  1. Successful Authentication
  2. Endpoint: /auth
  3. Validates login with correct credentials
  4. Checks for 200 status code

  5. Verifies access token generation

  6. Failed Authentication

  7. Endpoint: /auth
  8. Validates login with incorrect password
  9. Checks for 401 unauthorized status

Test Scenarios

  1. Get All Roles
  2. Endpoint: GET /roles
  3. Validates response status code

  4. Checks for list of roles

  5. Create Role

  6. Endpoint: POST /roles
  7. Validates role creation

  8. Checks response status and content

  9. Update Role

  10. Endpoint: PATCH /roles/{role_id}
  11. Creates a role
  12. Updates role name

  13. Validates update process

  14. Delete Role

  15. Endpoint: DELETE /roles/{role_id}
  16. Creates a role
  17. Deletes the role
  18. Validates deletion process

User Router Tests (test_user_router.py)

Test Scenarios

  1. Get All Users
  2. Endpoint: GET /users
  3. Validates response status code

  4. Checks for list of users

  5. Create User

  6. Endpoint: POST /users
  7. Creates user with a specific role

  8. Validates user creation

  9. Update User

  10. Endpoint: PATCH /users/{user_id}
  11. Creates a user
  12. Updates user name

  13. Validates update process

  14. Delete User

  15. Endpoint: DELETE /users/{user_id}
  16. Creates a user
  17. Deletes the user
  18. Validates deletion process

Testing Approach

  • Uses pytest and anyio for asynchronous testing
  • Leverages AsyncClient for API route testing
  • Provides comprehensive coverage of CRUD operations
  • Ensures proper authentication and authorization

Test Database Management

  • Temporary database created for each test session
  • Automatically dropped after test completion
  • Ensures clean, isolated testing environment
